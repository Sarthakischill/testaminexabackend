2. THE CORE ISSUE
================================================================================

The entire frontend is built but has ZERO backend integration. Every interactive
element (login, register, add to cart, checkout, COA lookup, account management)
is purely visual. The site cannot function as a real storefront without:

  a) An authentication system to gate access
  b) An order/cart system to handle purchases
  c) A payment integration to process transactions
  d) A data layer to serve products from a real source

The client's existing infrastructure is on Squarespace, but Squarespace does NOT
support headless commerce (no cart/checkout API for external frontends). This
means we cannot simply "plug in" Squarespace as a backend.

The client requires TWO payment methods:
  1. Credit card via QuickBooks Online (all CC payments must sync to their QBO)
  2. Zelle (manual payment with instructions displayed to the user, 10% discount)

This is a non-standard payment setup that rules out typical checkout solutions
(Shopify checkout, Stripe Checkout, Squarespace checkout redirect).

3. OPTIONS & DIRECTIONS
================================================================================

OPTION A: Full Custom Backend with Supabase
-------------------------------------------
Build everything custom using Supabase as the backend.

  Auth:         Supabase Auth (email/password, session management)
  Database:     Supabase Postgres (users, products, orders, cart, COA batches,
                referral codes, commission tracking)
  Storage:      Supabase Storage (COA PDFs, any file uploads)
  Products:     Stored in Supabase products table (only 4 SKUs now, easy to
                manage; expandable for pens and tabs later)
  Cart:         Custom cart system (Supabase table or client-side state)
  Orders:       Custom order table tracking status, items, payment method,
                user, referral code
  Payments:     QuickBooks Online API for credit card processing
                + Zelle instructions displayed in-app (collect name, email,
                phone to correlate manual Zelle payments)
  Route Guard:  Next.js middleware.ts redirecting unauthenticated users
  COA Data:     Supabase table with batch records + PDF links

  Pros:
  - Full control over every feature
  - Single auth system (no split between Squarespace and custom)
  - Seamless UX (no redirects to external sites)
  - Handles the QBO + Zelle payment model natively
  - Easy to add commission tracking, referral codes, order history
  - Scales to support pens, tabs, and protocol bundles later

  Cons:
  - More upfront development work
  - Product/inventory management is manual (no Squarespace sync)
  - Need to integrate QuickBooks Online API (requires OAuth, API key)
  - Zelle payments require manual verification by the client team


OPTION B: Hybrid (Supabase Auth + Squarespace Commerce)
-------------------------------------------------------
Use Supabase for auth/gating only, redirect to Squarespace for checkout.

  Auth:         Supabase Auth
  Products:     Pull from Squarespace Commerce API (read-only) or hardcode
  Cart:         Client-side state (context + localStorage)
  Checkout:     Redirect to Squarespace product pages for payment
  Orders:       Pull order history from Squarespace Commerce API by email
  COA Data:     Supabase

  Pros:
  - Less custom backend work
  - Uses existing Squarespace order management

  Cons:
  - Checkout redirect breaks the premium UX
  - Two separate systems (Supabase for auth, Squarespace for commerce)
  - Cannot handle QuickBooks/Zelle payment split natively
  - Squarespace Commerce API is read-only (no headless cart/checkout)
  - Customer has two "accounts" (portal login vs Squarespace account)


OPTION C: Squarespace Customer Accounts + Embedded Commerce
------------------------------------------------------------
Try to leverage Squarespace's built-in customer accounts and embed their
commerce widgets into the custom site.

  Pros:
  - Minimal custom backend

  Cons:
  - Squarespace customer accounts are tightly coupled to their hosted site
  - Cannot be embedded in an external Next.js app
  - Does not support the QBO/Zelle payment model
  - Defeats the purpose of a custom build
  - NOT RECOMMENDED


RECOMMENDATION: OPTION A (Full Custom with Supabase)
-----------------------------------------------------
Given the requirements (gated access, custom QBO + Zelle payment flow,
commission tracking, single seamless UX), Option A is the only approach
that actually delivers what the client needs. The non-standard payment
setup (QuickBooks credit card + Zelle with 10% discount) cannot be handled
by Squarespace or any standard checkout redirect.
